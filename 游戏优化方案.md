## 二、核心问题清单

### 🔴 P0 · 阻塞/安全

- **P0-1 好友房对局未落 `TcgMatch`**：当前好友房打完一局，**不写入 `TcgMatch` 表 / 不更新 ELO**。排行榜不会变化。
- **P0-2 服务端无权威校验**：好友房动作 API 只是追加到 action log，没有用 `applyAction` 校验合法性 —— 可被直接伪造。
- **P0-3 房间无 TTL 清理**：`TcgRoom` 无过期逻辑，废弃房间永久占用 6 位码空间，**长期会冲突**。

### 🟠 P1 · 体验/功能

- **P1-1 轮询频率固定 1s**：对战中也一直 1s 拉取；**无动作时应指数退避** → 节省 30-50% 请求。
- **P1-2 断线重连未实现**：刷新页面或掉线后，即使房间还在服务端，也回不去对局。
- **P1-3 `TcgDeck` 与好友房解耦**：玩家无法选择保存的自定义卡组进好友房 —— 只能预设 / 临时自选。
- **P1-4 `deck` 页面入口孤岛**：卡组构筑器 `/game/deck` 没在导航里，用户找不到。
- **P1-5 结束对战 UX**：好友房没有「再来一局」「换对手」；只能返回大厅。
- **P1-6 等待对手超时**：创建房间后对手一直不加入，没有超时提示。

### 🟡 P2 · 优化/打磨

- **P2-1 `preview` 路由用途不明**：未在导航中出现，疑似遗留实验页。
- **P2-2 `types.ts` 中残留 `/game/play` 注释**：已删除但注释未清理。
- **P2-3 日志中 action 体量膨胀**：一局 30 回合动作量可达 150+，`state` JSON 会很大 → 建议分表 `TcgRoomAction`。
- **P2-4 无观战 / 战报回放**：`TcgMatch.replay` 已预留字段但无前端页面。
- **P2-5 AI 难度单一**：`ai.ts` 无难度分级，练习体验固定。
- **P2-6 音效仅有基础**：出牌/受击/胜负 6 种，组合技触发时无特效。

### 🟢 P3 · 技术债

- **P3-1 Prisma 使用 `(prisma as any)`**：好友房 API 里使用类型断言绕过，需补全类型。
- **P3-2 `Battle.tsx` 单文件 ~1100 行**：职责过重，建议拆分为 `BattleStage` / `HandArea` / `EventSlots`。
- **P3-3 无 E2E 测试**：只有引擎单测，好友房端到端流程无自动化测试。

## 三、优化方案（按优先级排序）

### 🔴 Phase A：安全 & 数据一致性（2-3 天）

**A1. 服务端权威校验（P0-2）**
- `POST /api/tcg/room/action` 收到动作后，用 `initGame(seed, decks)` + 历史动作重放 → 验证新动作合法 → 再 append。
- 伪代码：
  ```ts
  const state = rebuildState(room); // seed + actions 重放
  try { applyAction(state, newAction); }
  catch { return fail('非法动作'); }
  ```

**A2. 好友房对局落库（P0-1）**
- 游戏结束时（`ended = true`）：
  - 写入 `TcgMatch`（mode='friend', replay=actions, winnerId）
  - 更新双方 `TcgPlayer.rating` 用 ELO 公式
  - 推送排行榜失效

**A3. 房间 TTL 清理（P0-3）**
- `TcgRoom` 加 `expiresAt`（创建时 +30min）
- Cron 脚本：`scripts/cleanup-rooms.ts` 每 10min 删除过期 `waiting` 房间
- 或用 Next.js Route Handler 按需清理

---

### 🟠 Phase B：体验完善（3-4 天）

**B1. 轮询指数退避（P1-1）**
- 无新动作时 1s → 2s → 4s（上限 5s）；有新动作重置为 1s
- 引入 `visibilitychange` 监听：标签隐藏时暂停轮询

**B2. 断线重连（P1-2）**
- `/api/tcg/room/status` 已支持返回 `seed/firstPlayer/decks/actions`
- 前端：页面挂载时检查 `sessionStorage` 的 roomCode，存在则请求 `/status` 重建引擎并继续

**B3. 自定义卡组接入（P1-3）**
- `/game/room` 菜单中增加「我的卡组」tab（调 `GET /api/tcg/decks`）
- 将 `/game/deck` 加入顶部导航

**B4. 对局收尾 UX（P1-5, P1-6）**
- 结束弹窗增加「再来一局」：双方同意后复用房间（重置 state + actions）
- 等待对手 10 分钟无人加入 → 提示「邀请码可能失效，重新创建」

---

### 🟡 Phase C：内容扩展（5-7 天）

**C1. AI 难度分级（P2-5）**
- `ai.ts` 接入 `difficulty: 'easy' | 'normal' | 'hard'`
  - easy：随机合法动作
  - normal：贪心优选（最高伤害/最低成本）
  - hard：2 层 minimax，对手模拟

**C2. 战报回放（P2-4）**
- `/game/replay/[matchId]` 新页，用 `TcgMatch.replay` 反序列化
- 可暂停/快进/步进，复用 `Battle.tsx` 只读模式

**C3. 观战（v2 目标）**
- `/game/spectate/[code]`：只读 watch 好友房
- 需要服务端对 guest 权限以外的用户允许 `status` 只读访问

**C4. 赛季 & 奖励（P2-4 扩展）**
- 激活 `TcgSeason`，天梯赛季重置 rating → 1000
- 赛季末发放限定卡（复用 `TcgCollection`）

---

### 🟢 Phase D：技术债（1-2 天）

**D1. 移除 `(prisma as any)`（P3-1）**
- 运行 `npx prisma generate`，确认类型已生成
- 全局替换为 `prisma.tcgRoom.*`

**D2. `Battle.tsx` 拆分（P3-2）**
- `Battle.tsx`（容器，~300 行）
  - → `BattleStage.tsx`（战场 + 事件槽）
  - → `HandArea.tsx`（手牌）
  - → `PlayerBar.tsx`（头像/流量/技能）
  - → `BattleOverlay.tsx`（换牌弹窗/结束弹窗）

**D3. E2E 测试（P3-3）**
- Playwright：创建房间 → 加入 → 一局流程 → 验证落库
