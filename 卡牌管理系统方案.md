# 🎛️ 陈泽卡牌 TCG — 独立管理系统方案 v1

> 定位：为 `/game/*` 卡牌对战模块配套的**独立运营后台**，与社区 `/admin` 完全分离。
> 路由：`/tcg-admin/*` · 独立登录 / 独立权限 / 独立侧边栏 / 独立主题。
> 目标：让运营人员 **无需改代码**即可维护卡池、平衡性、赛季、玩家数据、战报与作弊处理。

---

## 一、为什么要独立

| 维度 | 社区 `/admin` | TCG `/tcg-admin` |
|------|:-------------:|:----------------:|
| 用户模型 | 粉丝 / 助理 / 董事长 | 玩家（拥卡 / 段位 / 卡组） |
| 内容主体 | 帖子 / 相册 / 活动 | 卡牌 / 卡组 / 战报 / 赛季 |
| 角色岗位 | 内容审核 / 运营 | **游戏策划 / 数值 / 赛事** |
| 风险模型 | 违规内容 | **作弊 / 刷分 / 平衡性崩坏** |
| 未来拆分 | 主域名 | 可抽离 `tcg.xxx.com` 子域名独立部署 |

**结论**：两套后台受众、工具、权限完全不同，混在一起会让两边都很难用。独立后 TCG 模块可单独开发、单独发版、单独下线。

## 四、数据模型（Prisma 新增 8 张表）

> 全部加到现有 `prisma/schema.prisma`，表名前缀 `Tcg*` 避免与社区表混淆。

```prisma
// ==================== TCG · 卡牌数据 ====================

model TcgCard {
  id          String   @id                    // C01 / I08 / E12 等固定编号
  name        String
  type        String                           // character | item | equipment | effect | event
  subtype     String?                          // instant | weapon | armor
  rarity      String                           // N | R | SR | SSR
  cost        Int      @default(0)
  attack      Int?
  health      Int?
  description String                           // 规则描述（含【关键字】）
  flavor      String?
  imagePath   String?                          // /cards/*.png 或 COS URL
  skillKeys   String   @default("[]")          // JSON array，对应 engine 技能注册表
  keywords    String   @default("[]")          // JSON array：taunt / charge / stealth ...
  seasonId    String?                          // 限定赛季（null = 通用池）
  status      String   @default("active")      // active | disabled | draft
  sortOrder   Int      @default(0)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([type])
  @@index([rarity])
  @@index([status])
  @@index([seasonId])
}

model TcgDeckPreset {
  id          String   @id @default(cuid())
  name        String                           // 「铁粉嘲讽」「连击流量」...
  description String
  archetype   String                           // aggro | control | combo
  cardIds     String                           // JSON: [{id, count}]
  isStarter   Boolean  @default(false)         // 新手起步卡组
  status      String   @default("active")      // active | disabled
  sortOrder   Int      @default(0)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([isStarter])
  @@index([status])
}

// ==================== TCG · 玩家资产 ====================

model TcgPlayer {
  userId      String   @id                     // 与现有 User.id 一致
  rating      Int      @default(1000)          // ELO 分
  tier        String   @default("iron")        // iron | silver | gold | diamond | master
  energy      Int      @default(5)              // 当日剩余对战能量
  energyDate  String                            // YYYY-MM-DD 重置
  wins        Int      @default(0)
  losses      Int      @default(0)
  banStatus   String   @default("normal")       // normal | warned | banned
  banUntil    DateTime?
  banReason   String?
  lastPlayAt  DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([tier])
  @@index([banStatus])
  @@index([lastPlayAt])
}

model TcgCollection {
  id        String   @id @default(cuid())
  userId    String
  cardId    String
  count     Int      @default(1)                 // 持有数量
  shards    Int      @default(0)                 // 碎片（10 碎 = 1 张）
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@unique([userId, cardId])
  @@index([userId])
}

model TcgDeck {
  id        String   @id @default(cuid())
  userId    String
  name      String
  cardIds   String                                // JSON: [{id, count}]
  isActive  Boolean  @default(false)              // 当前出战卡组
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@index([userId, isActive])
}

// ==================== TCG · 对局 ====================

model TcgMatch {
  id          String   @id @default(cuid())
  mode        String                               // practice | ranked | friend | boss
  seasonId    String?
  playerAId   String                                // User.id
  playerBId   String?                               // null = AI / Boss
  deckA       String                                // JSON 快照
  deckB       String                                // JSON 快照
  replay      String                                // JSON action 序列（engine 可重放）
  winnerId    String?
  ratingDelta Int      @default(0)                  // A 的 ELO 变化量
  turns       Int      @default(0)
  durationSec Int      @default(0)
  endedReason String?                              // hp_zero | turn_15 | surrender | timeout
  createdAt   DateTime @default(now())

  @@index([playerAId, createdAt])
  @@index([playerBId, createdAt])
  @@index([seasonId])
  @@index([mode])
}

model TcgSeason {
  id          String   @id @default(cuid())
  name        String                                // 「S1 · 流量纪元」
  startAt     DateTime
  endAt       DateTime
  status      String   @default("draft")           // draft | ongoing | settling | closed
  rewardRules String                                // JSON：段位 → 奖励配置
  featuredCards String @default("[]")               // JSON: 本季限定卡 id 列表
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([status])
}

// ==================== TCG · 运营 ====================

model TcgOperator {
  id        String   @id @default(cuid())
  email     String   @unique
  password  String
  name      String
  role      String   @default("tcg_editor")        // tcg_super | tcg_ops | tcg_editor
  isActive  Boolean  @default(true)
  lastLogin DateTime?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@index([role])
}

model TcgAuditLog {
  id         String   @id @default(cuid())
  operatorId String
  action     String                                 // card.update / player.ban / season.settle ...
  targetType String?
  targetId   String?
  before     String?                                // JSON
  after      String?                                // JSON
  ip         String?
  createdAt  DateTime @default(now())

  @@index([operatorId, createdAt])
  @@index([action])
  @@index([targetType, targetId])
}
```

**迁移策略**（可回滚）：
1. 新增表全部带 `Tcg` 前缀，不改任何现有表 → 回滚 = 单独删 migration
2. `Card` 数据通过 `prisma/seed.ts` 从 `cardPresets.ts` 导入，旧文件**保留**作为兜底
3. 前台 `/game/*` 改为优先读 DB、DB 空时 fallback `cardPresets.ts` —— 平滑过渡

## 五、权限体系

**独立表 `TcgOperator`**，不复用 `Admin`：

| 角色 | `role` 值 | 权限 |
|------|-----------|------|
| TCG 超管 | `tcg_super` | 全部权限 + 运营账号管理 + 审计日志 |
| TCG 运营 | `tcg_ops` | 卡池编辑 / 赛季 / 补偿 / 封禁 / 举报 |
| TCG 策划 | `tcg_editor` | 只读 + 卡池草稿编辑（需审核发布） |

**鉴权**：
- 复用现有 `@/lib/auth` 的 JWT 框架，但使用**独立 cookie** `tcg_admin_session`
- 中间件 `src/middleware.ts` 已有 `/admin` 保护，追加 `/tcg-admin` 独立校验
- 所有 `/api/tcg/admin/*` 请求必须带 `tcg_admin_session`，写操作统一走 `TcgAuditLog` 落日志

## 八、关键技术决策

### 8.1 卡池数据从代码迁 DB

**痛点**：现在 `cardPresets.ts` 改一个数值要发版；管理后台必须接数据库。

**迁移方案**：
1. `prisma migrate dev --name add_tcg_tables` 建表
2. `prisma/seed.ts` 追加 `seedTcgCards()`，把 `cardPresets.ts` 40 张写入
3. 前台改读 `/api/tcg/public/cards`（带 5 分钟缓存）
4. 本地兜底：如果 API 空数据，继续使用 `cardPresets.ts`（防止 DB 故障炸前台）

### 8.2 战报回看

- 战报 JSON 格式：`{ seed, deckA, deckB, actions: [{turn, player, action, payload, timestamp}] }`
- 回看页调用 **同一份** `engine.ts` 纯函数，`reduce` action 到每一帧
- 支持前进 / 后退 / 跳转到任意回合
- 对可疑作弊战报，可点击"重算校验"，看服务端记录的 state 与客户端一致性

### 8.3 运营操作审计

所有写操作走中间件拦截，统一落 `TcgAuditLog`：
- `before` / `after` 存 JSON diff
- 管理员误操作可从日志找到旧值手动回滚
- 对 SSR 卡改动、封号、补偿等敏感操作**强制留痕**

### 8.4 独立登录 / 独立 cookie

- 现有社区 Admin 登录 cookie 名：`admin_session`
- TCG 运营登录 cookie 名：**`tcg_admin_session`**（独立 secret，互不干扰）
- 同一浏览器可同时持有两套 session（运营既是社区管理员又是 TCG 运营时无需反复登录）

## 十、回滚策略

用户一贯要求"可回滚"，本方案执行时每个提交独立可 `git revert`：

| 提交粒度 | 可回滚范围 |
|---------|-----------|
| 单次提交 = 1 张表 migration + 对应 API + 对应页面 | 回滚该模块不影响其他 |
| Prisma migration 可用 `prisma migrate reset` + 保留前向 migration 文件 | 数据库层可回滚 |
| 前台读 DB 失败时 fallback 到 `cardPresets.ts` | DB 故障不影响玩家 |
| 独立路由 `/tcg-admin` | 整体回滚只需删 `(tcg-admin)` 路由组 + 相关 API 目录 |

## 十一、风险与对策

| 风险 | 等级 | 对策 |
|------|:----:|------|
| 卡池迁 DB 后前台爆炸 | 高 | fallback `cardPresets.ts`；DB 读失败降级兜底 |
| 管理员误删卡牌 | 中 | 软删除（`status=disabled`）+ 审计日志保留 before |
| 补偿发放被滥用 | 中 | 金额 / 数量上限 + 强制填写发放理由 + 走审计 |
| 战报数据膨胀 | 中 | 战报 90 天归档到冷存，仅保留近 30 天热数据 |
| TCG 运营乱改数值 | 中 | 草稿 / 发布分离，SSR 改动需 `tcg_super` 二次审核 |
