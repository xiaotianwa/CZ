# 🎴 陈泽传媒卡牌游戏 — 可行性设计方案 v2

> 依托已有「陈泽」粉丝社区（Next.js 14 + Prisma + libSQL/Turso + TailwindCSS）扩展的轻量化卡牌小游戏。
> 定位：**社区内置免费小游戏**，单局 3-10 分钟，提升日活、内容消费与社区黏性。
> **完全免费 · 永不售卖卡牌**。

---

## 一、核心设计目标

| 目标 | 说明 |
|------|------|
| **强 IP 绑定** | 所有卡牌、技能、剧情均来自「陈泽传媒」内容（短视频梗、经典台词、合作艺人、直播名场面等） |
| **轻量易上手** | 单局 ≤ 3 分钟，规则 30 秒看懂，移动端优先 |
| **复用社区资产** | 账号、积分、等级、关注关系、CDN 图片、Prisma 数据层一并复用 |
| **可持续运营** | 支持周期性发布新卡包（对应陈泽新作品/新动态），形成内容 → 游戏 → 回流社区的闭环 |
| **完全免费** | 不设任何付费点，无抽卡、无皮肤售卖、无月卡，规避版号/赌博合规问题，纯靠社区积分循环 |

## 二、世界观 & 题材

**主题**：「陈泽传媒宇宙 · 造星计划」
粉丝扮演**经纪人**，用陈泽 IP 旗下的艺人/梗/名场面组成卡组，参加每日「流量擂台」对战，争夺榜单与限定卡。

### 卡牌分类（4 类 × 4 稀有度）

| 类型 | 含义 | 示例（可替换为真实内容） |
|------|------|--------------------------|
| 🎤 **艺人牌（Character）** | 陈泽本人、旗下/合作艺人 | 「主播·陈泽」「搭档A」 |

## 三、玩法规则（MVP 版）

### 3.1 基础对战（1v1 · 单局 3-10 分钟）

- 每位玩家携带 **25 张卡组**（四类至少各 2 张）
- 双方初始「流量值」**40**，目标：把对方流量打到 0，或 **15 回合**结束时流量更高者胜
- 每回合抽 1 张，消耗「能量」出牌（能量每回合 +1，上限 **10**）
- **爆点回合**：从第 8 回合起所有事件牌伤害 +1，逼出终局防拖沓
- **对战形态**：
  - MVP：**异步 PVP**（对手为真人卡组快照 + AI 托管）+ 单人 Boss 挑战
  - v1.1：**好友房**（6 位邀请码，URL 分享，HTTP 轮询，无需 WebSocket）
  - v2：实时房间（按需再引入 WebSocket/PartyKit）

### 3.2 核心机制（差异化点）

| 机制 | 玩法 |
|------|------|
| **组合技** | 特定艺人 + 梗组合触发「名场面」特效（爆发伤害 + 播放对应短视频片段 3s） |
| **流量发酵** | 事件牌进入场上后每回合叠加热度，3 回合后结算造成 AOE |
| **粉丝应援** | 对局中社区其他在线粉丝可花 1 积分为任一玩家「点赞」，增加 1 点流量（限额） |
| **每日挑战** | AI Boss（如「黑粉军团」「友商主播」）每天 3 次，通关掉卡片碎片 |

### 3.3 获取 & 成长（完全免费）

- **无任何付费点**，卡片 100% 免费获取，来源：
  1. 社区活跃任务（发帖/评论/签到）→ 积分 → 兑换指定卡
  2. 每日挑战掉落碎片 → 合成
  3. 周赛/月赛排名奖励限定卡（荣誉性，非数值碾压）
  4. 看「陈泽传媒」新内容触发解锁（埋点：看完某条动态 > 10s 送碎片）← **核心回流机制**

### 3.4 能量经济（防挂机刷分）

- 每日免费 **对战能量 5 点**，每局消耗 1 点
- 用完后可用社区积分 **1:1 兑换**（积分来自发帖/评论/点赞被赞）
- 不设任何现金购买入口

## 四、技术可行性

### 4.1 完全可在现有栈内实现

| 需求 | 复用 | 新增 |
|------|------|------|
| 账号/登录 | ✅ 现有 `users` + NextAuth/自研 JWT | — |
| 卡图/特效资源 | ✅ 腾讯云 COS（已接入 `cos-nodejs-sdk-v5`） | 卡图/音效上传规范 |
| 数据存储 | ✅ Prisma + libSQL（`@libsql/client`） | 游戏相关 5 张表（下文） |
| 实时对战 | ❌ 现无 WebSocket | MVP 异步 PVP + 好友房 HTTP 轮询（1.5s/次），v2 再引入 WebSocket |
| 动效渲染 | TailwindCSS + CSS/Framer Motion | 卡牌翻转/释放用 CSS transform + GPU 合成，够用 |
| 图表 | ✅ 已有 `echarts` | 用于赛季数据展示 |
| 校验 | ✅ 已有 `zod` | 出牌动作服务端校验 |

### 4.2 架构分层

```
┌─ src/app/game/                 前台页面（卡组/对战/商店/排行）
├─ src/app/api/game/*            游戏 REST API（Next.js Route Handlers）
├─ src/lib/game/
│   ├─ engine.ts                 纯函数战斗引擎（同一份代码前后端共用）
│   ├─ cards.ts                  卡牌数据 & 技能注册表
│   ├─ matchmaking.ts            异步匹配（按段位选取快照）
│   ├─ room.ts                   好友房：code 生成、state 版本化、轮询 diff
│   └─ economy.ts                积分/碎片/合成/能量规则
├─ src/app/admin/game/           管理后台：卡池编辑、赛季配置、封禁作弊
└─ prisma/schema.prisma          新增 5 张表
```

**关键原则**：战斗引擎为**纯函数**（输入 state + action → 新 state），客户端预演动画，服务端权威判定防作弊。

### 4.3 数据模型（新增 Prisma 表）

```prisma
model Card {
  id          String   @id
  name        String
  type        String   // character | meme | work | event
  rarity      String   // N | R | SR | SSR
  cost        Int
  attack      Int
  health      Int
  skillKey    String?  // 对应 engine 中的技能实现
  artUrl      String   // COS 地址
  seasonId    String?
  createdAt   DateTime @default(now())
}

model UserCard {
  id        String @id @default(cuid())
  userId    String
  cardId    String
  count     Int    @default(1)
  shards    Int    @default(0)
  @@unique([userId, cardId])
}

model Deck {
  id        String @id @default(cuid())
  userId    String
  name      String
  cardIds   String   // JSON 数组
  isActive  Boolean  @default(false)
  updatedAt DateTime @updatedAt
}

model Match {
  id         String   @id @default(cuid())
  playerAId  String
  playerBId  String   // 可为快照玩家
  replay     String   // JSON，引擎可重放
  winnerId   String?
  seasonId   String
  createdAt  DateTime @default(now())
}

model SeasonRank {
  userId    String
  seasonId  String
  points    Int    @default(0)
  wins      Int    @default(0)
  losses    Int    @default(0)
  @@id([userId, seasonId])
}

model Room {
  id        String   @id @default(cuid())
  code      String   @unique    // 6 位邀请码
  hostId    String
  guestId   String?
  state     String              // JSON 战局状态
  version   Int      @default(0) // 客户端轮询比对
  status    String   @default("waiting") // waiting | playing | finished
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

model DailyEnergy {
  userId   String
  date     String   // YYYY-MM-DD
  used     Int      @default(0)
  @@id([userId, date])
}
```

### 4.4 反作弊要点

1. **出牌动作走 API**，服务端持有权威 state，客户端仅提交 `action`。
2. **战报可重放**：`Match.replay` 存 action 序列，随时重算校验。
3. **积分变动**全部通过事务 + 速率限制（现有中间件已具备基础设施）。

## 五、UI / 交互建议

- **入口**：社区顶部导航新增「🎴 擂台」入口；用户中心新增「我的卡牌」Tab。
- **对战页**：移动端竖屏，上下双方区域，中间出牌轨道；卡牌用 `aspect-[3/4]` + 圆角渐变边框。
- **卡图规格**：`750×1050` webp，≤ 120KB，走 COS + `next/image`。
- **暗色模式**：沿用社区已有主题 token，不再单独做。
- **动效预算**：单局 JS 主线程占用 < 30%，低端机降级为纯淡入淡出。

## 九、结论

> **结论：可行**。游戏可完全内生于现有 Next.js + Prisma + libSQL 技术栈，**零新增基础设施**（不需要 WebSocket、不需要游戏服务器），MVP 预计 **~3 周** 可上线。
> 关键成功要素是 **内容强绑定** 与 **积分经济闭环**——让玩家为了陈泽的新内容回到社区，而不是为了游戏离开社区。
